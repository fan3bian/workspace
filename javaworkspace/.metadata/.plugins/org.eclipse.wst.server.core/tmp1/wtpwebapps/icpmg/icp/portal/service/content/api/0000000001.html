deployVirtualMachine

创建虚机

描述

  • 创建虚机过程第一步选择zone，调用listZones接口

  • 选择从模板或者ISO创建

  • 选择模板/ISO，调用listTemplates或者listIsos

  • 选择一种计算方案(serviceoffering)。计算方案包括了CPU内核数、CPU MHz、存储类型(share/local)、内存大小、网络速率限制、磁盘读写速率限制、存储标签、主机标签等。调用接口listServiceOfferings。  

    创建计算方案调用接口createServiceOffering  

  • 选择一种磁盘方案(diskoffering)。磁盘方案包括了存储类型、磁盘大小等。当以模板方式创建虚机时，磁盘方案选择的是数据盘。当以ISO方案时，磁盘方案为系统盘+数据盘。调用接口listDiskOfferings。创建磁盘方案调用接口listDiskOfferings

  • 选择一个网络。网络根据网络方案(networkoffering)建立。调用接口listNetworks。

  • 输入虚机显示名称

  • 调用接口deployVirtualMachine

响应举例+异步任务


响应则分两种，只要创建虚机的消息发送成功，不论执行是否成功，都会返回一个成功的消息，携带了执行创建虚机任务的ID 


{ 
"deployvirtualmachineresponse": {"id":"cafd38b6-b6f1-46c6-b9ea-62d0ab47f8cb",
"jobid":"435c9d63-fd68-4f64-9692-c1c4387cbdb8"
} 
}


后面如果没有任何操作，即便虚机创建成功，也不会有任何响应。这是因为虚机创建任务为异步执行方式，需要调用任务查询接口查询任务执行情况，参数即为返回的jobid。可设置定时器，每隔五秒中调用任务查询接口查询任务执行情况。 


http://localhost:8080/client/api
?sessionkey=fqZ6TaJLD3b9LQnffhV5x81CBBU%3D
&command=queryAsyncJobResult
&response=json
&jobid=435c9d63-fd68-4f64-9692-c1c4387cbdb8
